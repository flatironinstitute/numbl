# Standard Library

The standard library is the set of MATLAB-surface functions that numbl implements in MATLAB itself rather than as TypeScript `IBuiltin`s. These are thin wrappers, compatibility helpers, and higher-level utilities where writing the logic in MATLAB is more natural than hard-coding it in the interpreter.

Examples include `inputParser`, `readmatrix`, `decomposition`, and a collection of namespace shims (e.g., under a `+matlab/+internal/` search path).

## Why it is bundled

The CLI can read `.m` files from disk, but the browser web app cannot. To make the same stdlib available in both, the stdlib files are embedded into the core at build time as a single generated TypeScript module. The module exports arrays of `{ name, source }` pairs that `executeCode` injects into every run's `LoweringContext` as pre-registered workspace files.

There are two arrays:

- **flat stdlib** — functions and classes that live on the default search path.
- **shims** — compatibility wrappers that need to be resolved through a search-path mechanism (e.g., namespace packages).

## Regenerating the bundle

The bundle is regenerated by a dedicated script. The build scripts run it automatically; running the TypeScript source directly (without building first) requires running the script manually after any stdlib change. The generated file is not hand-edited.

See `CONTRIBUTING.md` for the current command and workflow.

## Adding a stdlib function

1. Decide between stdlib (`.m`) and builtin (`IBuiltin`). Rough guide: if the behavior is naturally expressible in MATLAB and not performance-critical in the inner loop, use a `.m` file. If it is a primitive or needs tight JIT integration, make it a builtin.
2. Add the `.m` file under the stdlib source location with the function or class name matching the file name.
3. Regenerate the bundle.
4. Add tests (unit or integration) and verify behavior matches MATLAB.
